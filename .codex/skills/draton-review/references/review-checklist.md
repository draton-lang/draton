# Review checklist

## Primary questions

- Can the change break parser, typechecker, runtime, CLI, or release behavior?
- Does the change violate canonical language rules or repository boundary rules?
- Are docs, examples, and tests still aligned with the implementation?
- Does the change weaken branch, commit, or release discipline?
- Did the author verify the most relevant path?

## Report shape

- Findings first, ordered by severity.
- Use concrete file references.
- Keep summaries brief and secondary.
- Call out missing tests or missing verification when relevant.
